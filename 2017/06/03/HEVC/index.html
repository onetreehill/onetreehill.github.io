视频编码浅谈（一）

发表于 2017-06-03   |     |   阅读次数

不出意外，外来十年应该就在这个行当了，如同前文所述，这个行当未来可期。所以为了给大家一个直观的印象，打算从这篇开始，介绍一下这份工作的内容，权当是我精炼总结的一个机会。

视频编码的目的在于压缩原始视频数据，以达到网络传输分享的目的。目前视频娱乐已经成为日常娱乐的核心，比如：体育赛事的直播、转博、点播；在线电影、电视剧、动漫；不同终端之间的直播互动等等。对视频清晰度的追求从未改变，从标清、720P、1080P，到现今的2K、4K，不仅推动了硬件产业的发展，也提出了对视频编码压缩技术的进一步需求。（强行找意义而已，可无视！）

好了，说正题。视频编码的总体思想在于利用视频帧在时间和空间上的高相关性，在上一帧，或者当前帧的某些空间区域的基础上传输变化量（残差）：

Frame_current = Frame_last + residual-plot，

再利用傅里叶变换的思想（实际中使用离散余弦变换DCT，只有实数部分，没有虚数部分）将residual-plot变换到频域，传输频域上的“主要量”（民科喜欢自造名词，无视无视！！）。“主要量”通过量化过程进行自动判断，很显然DCT之后将得到一个非整型的矩阵，并且低频区间的幅值要远大于高频区间的幅值，于是可以设定一个新的标度，将该矩阵重新标度，使之整数化，如此可获得的好处有：一，整形化的矩阵传输的数据量更小；二，幅值较小的可以被量化为0，这样使得矩阵稀疏化。当然，这种二次标度实质是一种舍弃局部信息的过程，所以会使图像失真（损失某些信息）。所以，量化过程也是视频编码质量损失的唯一来源。但是也是视频编码的核心环节。

现在稍微理理上述过程：

  1. 选择参考帧（Frame_last）以及模式（利用空间相关还是时间相关）；
  2. 得到残差信息（上述公式）；
  3. DCT变换；
  4. 量化；
  5. 传输。
    所以，视频编码的整个过程并不复杂，相反结构清晰，所有操作都服务于压缩数据量这一核心目标。
